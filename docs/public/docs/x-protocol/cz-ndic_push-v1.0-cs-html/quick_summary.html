Krátký souhrn¶

Data se zasílají od poskytovatele odběrateli.

  1. Odběratel musí provozovat HTTP server a musí poskytnout poskytovateli url (nazývané doručovací url), na které má poskytovatel data zasílat pomocí HTTP POST požadavku (doručovaná zpráva je jediným obsahem těla požadavku).
  2. Poskytovatel nakonfiguruje novou subskripci pro dané doručovací url.
  3. Poskytovatel na úvod odešle všechny zprávy, které byly vytvořeny v minulosti a jsou stále ještě platné.
  4. Ve chvíli, kdy nastanou změny relevantní pro daného odběratele, poskytovatel odešle změny odběrateli.

Poskytovatel musí odesílat zprávy v pořadí a v jednom vlákně.

Odběratel musí odpovědět stavovým kódem HTTP 200.

V případě, že nastanou problémy s doručením zprávy (doručovací url není dostupné nebo stavový HTTP kód nemá hodnotu 200), poskytovatel musí pokračovat v opakovaných pokusech zprávu doručit do doby, než uspěje.
