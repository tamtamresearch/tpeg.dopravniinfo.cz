Implementace pomocí HTTP POST¶

Role a scénáře¶

Odběratel musí zprovoznit webový server a poskytnout poskytovateli url, na které se mají data zasílat (tzv. doručovací url).

Poskytovatel zasílá zprávu pomocí HTTP POST s tím, že zpráva je jediným obsahem těla požadavku.

Varování

Neočekávejte multipart-form data nebo urlencoded form data!!!

Odběratel potvrdí přijetí zprávy tím, že vrátí HTTP status kód 200. Jakákoliv jiná hodnota je považována za požadavek o opakované zaslání téže zprávy.

Ostatní vlastnosti HTTP¶

autorizace:Autorizace se nepoužívá. Po dohodě je možno použít odmítání požadavků zaslaných z jiných než dohodnutých IP adres.
komprese:Odběratel nemůže přinutit poskytovatel k zasílání komprimovaných dat (s použitím content-encoding: gzip). Veškerý obsah je doručován tak jak je bez jakéhokoliv zakódování.
content type:Očekávaný content-type záleží na specifikaci formátu a dohodě a jeho určení je mimo rámec specifikace tohoto protokolu. Ve většině případů se používá content-type application/xml.
max. velikost:Maximální velikost zaslané zprávy je 200 kB. Lze dohodnout jinou velikost.
https:HTTP server na straně odběratele může používat https protokol.
