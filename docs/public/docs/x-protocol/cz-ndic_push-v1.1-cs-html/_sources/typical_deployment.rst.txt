===========================================================
Typický průběh zřízení a provozování odběru protokolem PUSH
===========================================================

.. uml::
   :caption: Zřízení a provoz odběru
   :align: center

    @startuml
    skinparam BoxPadding 15
    participant event_src as "Zdroj informací"
    box "Odesilatel"
        actor pub_admin as "Správce"
        boundary reg_sub as "Registr\nodběrů"
        database pub_db as "Databáze\nse záznamy"
        control pub_proc as "Proces\nzasílání"
    end box
    box "Odběratel"
        actor recv_admin as "Správce"
        boundary recv_url as "URL\nodběratele"
        database recv_db as "Databáze\nse záznamy"
    end box
    recv_admin -> reg_sub: Zřízení odběru
    pub_admin -> reg_sub: Schválení odběru
    reg_sub -> pub_proc: Konfigurace zasílání
    group Odeslání výchozí dávky právě platných zpráv
            loop n-krát
            pub_proc -> recv_url: Odeslání zprávy
            recv_url -> recv_db: uložení záznamů
            recv_url --> pub_proc: HTTP status 200: OK
            end
    end
    group Změna či aktualizace záznamu
            event_src -> pub_db: aktualiace
            pub_db -> pub_proc: nastala změna
            pub_proc -> recv_url: Odeslání (změnové) zprávy
            recv_url -> recv_db: uložení záznamů
            recv_url --> pub_proc: HTTP status 200: OK
    end
    @enduml

Implementace systému odběratele
===============================

Odběratel si implementuje webovou službu, která je schopna na URL odběratele (specifickém pro konkrétní odběr) přijímat HTTP POST požadavky s tím, že tělo tohoto požadavku je poskytovanou zprávou, kterou si uloží k dalšímu zpracování.

Tato webová služba se pak vystaví na veřejně dostupné adrese. Je žádoucí, aby byla doména zabezpečena pomocí TLS (tedy dostupná protokolem HTTPS) a aby implementovala autentizaci formou Basic authentication (dle RFC 7617). Odběratel si ověří, např. pomocí nástrojů jako je curl, že je služba dostupná a funkční. Pro další účely si poznačí cílové url a uživatelské jméno a heslo pro autentizaci.

Zřízení odběru
==============

Odběratel navštíví Registr odběrů, dle potřeby si zřídí účet, a přistoupí k vytvoření nového odběru. Pro zvolený zdroj, který nabízí požadovaný obsah a užívá protokol PUSH, zadá především url, na kterou bude poskytovatel data zasílat a uvede i uživatelské jméno a heslo. Tyto údaje odpovídají url a jménu a heslu, které je systém odběratele připraven obsloužit.

Aktivace zasílání
=================

Po aktivaci zřízeného odběru (což může v případě interního schvalování u poskytovatele chvíli trvat) zašle poskytovatel první dávku zpráv, které obsahují záznamy, které v daném okamžiku platí.

Průběžné zasílání změn
======================

Od této chvíle pak systém poskytovatele čeká na interní aktualizaci poskytovaného obsahu, jakmile toto nastane, pošle odběrateli novou zprávu.
