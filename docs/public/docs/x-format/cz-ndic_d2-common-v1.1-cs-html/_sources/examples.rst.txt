Příloha A: Příklady zpráv
=========================

accident.xml - nehoda dvou vozidel
==================================

Publikace obsahuje právě jednu zprávu o nehodě dvou vozidel, z nichž
aspoň jedno je nákladní vozidlo.

.. literalinclude:: ../../samples/accident.xml
   :language: xml

Kořenový element a obálka publikace
-----------------------------------

Kořenový element `d2LogicalModel` deklaruje, že se jedná o zprávu používající DATEX II.

Element `exchange` obsahuje identifikátor poskytovatele.

Element `payloadPublication` je obálkou pro přenos samotných informací,
obsahující především elementy `situation`, kterých může být mnoho (v našem případě je jeden).

Atribut `xsi:type="SituationPublication"` v elementu `payloadPublication`
určuje, že tato obálka je typu Publikace situace.

Element `feedType` uvádí typ publikace, uri schématu.

Element `publicationTime` uvádí čas vydání této publikace jako celku.

Element `publicationCreator` identifikuje tvůrce publikace.

Popis jedné konkrétní situace
------------------------------ 

Následující element `situation` se může opakovat a popisuje jednotlivé samostatné dopravní informace. Element `situationRecord` se v tomto případě vyskytuje v elementu `situation` právě jednou.

Element `situationVersionTime` uvádí čas vydání této verze situace jako celku.

Element `headerInformation` popisuje způsob nakládání s popisovanými informacemi.

Popis události (co se stalo)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Atribut `xsi:type="Accident"` elementu `situationRecord` určuje typ
události. Při pohledu do DATEX II modelu lze ověřit, že jde o označení
typu potomka třídy `SitationRecord` v této dědičné posloupnosti
`Accident` -> `TrafficElement` -> `SitationRecord`.

Element `probabilityOfOccurrence` hodnotou `certain` stanovuje, že se tato událost určitě stala.

Element `accidentType` určuje typ nehody, v tomto případe hodnota
`accidentInvolvingHeavyLorries` uvádí, že jde o nehodu s účastí
těžkého nákladního vozidla.

Element `vehicleInvolved` předchozí informaci částečně opakuje tím, že jako charakteristiku vozidla, které se nehody účastní uvádí `lorry`, tedy nákladní vozidlo.

Časová platnost
^^^^^^^^^^^^^^^

Časová platnost situace je určena ve strukturách `validity` odhadem časového intervalu platnosti hodnotami od `overallStartTime` do `overallEndTime`.

Poloha události
^^^^^^^^^^^^^^^

Poloha události je popsána v elementu `groupOfLocations` a to přímo
vnořeným popisem, není používáno odkazu na předem definované
polohy. Atribut `xsi:type="Linear"` uvádí, že jde o lineární typ polohy
a ten je zde vyjádřen dvěma popisy. Nejprve pomocí lokačních tabulek v elementu `alertCLinear` typu `AlertCMethod2Linear`, poté pomocí odkazů do sítě Global Network v elementu `globalNetworkLinear`.

Všechny použité metody popisu polohy vyjadřují tutéž polohu. Vzhledem k tomu, že poloha v Lokačních tabulkách leží mezi dvěma body, tedy na úseku, musel být zvolen typ lokace `Linear`, s důsledkem, že `startPoint` a `endPoint` v `globalNetworkLinear` budou stejné.

Další obsah
^^^^^^^^^^^

Mimo časovou platnost situace dokument uvádí čas vydání první verze `situationRecordCreationTime` a aktuální verze `situationRecordVersionTime` záznamu `situationRecord`.

Element `source` uvádí tvůrce této informace, tedy kdo tuto informaci vytvořil a odpovídá za její správnost.

Element `generalPublicComment` obsahuje obecný popis děje popsaného ve zprávě, většinou jen v češtině. Jedná se o volný popis situace jejím tvůrcem.

brokendownvehicles.xml - dvě porouchaná nákladní vozidla
========================================================

Publikace obsahuje právě jednu zprávu o dvou porouchaných nákladních
vozidlech, které zcela blokují provoz po dané komunikaci.

.. literalinclude:: ../../samples/brokendownvehicles.xml
   :language: xml

Obálka publikace, časová platnost a poloha
------------------------------------------

Obálka publikace, časová platnost a poloha používají struktury shodné s předchozím příkladem.

Rozdíl je výhradně v popisu jedné konkrétní situace v elementu `situationRecord`.

Popis události (co se stalo)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Atribut `xsi:type="VehicleObstruction"` elementu `situationRecord` určuje typ události. Při pohledu do DATEX II modelu lze ověřit, že jde o označení
typu potomka třídy `SitationRecord` v této dědičné posloupnosti
`VehicleObstruction` -> `Obstruction` -> `TrafficElement` -> `SitationRecord`.

Element `probabilityOfOccurrence` hodnotou `certain` stanovuje, že se tato událost určitě stala.

Element `impact` uvádí v `trafficConstrictionType`, že důsledkem je úplná
neprůjezdnost komunikace.

Element `numberOfObstructions` hodnotou `2` uvádí, že existují dvě překážky provozu.

Element `vehicleObstructionType` hodnotou `brokenDownVehicle` vymezuje typ překážky na "porouchaná vozidla".

Element `obstructingVehicle` předchozí informaci doplňuje hodnotou `lorry` a tím upřesňuje, že jde o nákladní vozidlo nebo vozidla.

Časová platnost
^^^^^^^^^^^^^^^

Viz předchozí příklad.

Poloha události
^^^^^^^^^^^^^^^

Od předchozího příkladu se popis polohy liší v tom, že jsou použity 3 metody popisu polohy. Nejprve pomocí lokačních tabulek v elementu `alertCLinear` typu `AlertCMethod2Linear`, poté pomocí obecného popisu polohy na silnici v elementu `linearWithinLinearElement` a následně popsán pomocí odkazů do sítě Global Network v elementu `globalNetworkLinear`.

Nastává stejný případ jako v předchozím příkladu, s ohledem na lokaci ALERT-C která je liniová (nemůže být bodová) jsou zvoleny i ostatní metody jako liniové.

Všechny použité metody popisu polohy vyjadřují tutéž polohu.
