Koncepty¶

Tato kapitola popisuje základní koncepce, používané v tomto formátu. Detailní popis viz definice ve schématech.

Tato kapitola uvádí krátké ukázky ze skutečných XML dokumentů. Více ukázek viz Příloha A: Příklady zpráv.

Části XML dokumentu¶

XML Dokument sestává z následujících částí:

  • msg (document root plus some metadata)
  • detector (data from weather station)

Kořen dokumentu msg¶

Element msg představuje jednotlivou publikaci (jednu instanci přenášených dat).

Mimo element datetime lze ostatní elementy v záhlaví považovat za konstantní.

Metadata (detector)¶

Element detector nese data naměřená na jednom detektoru. Těchto elementů může být přítomno i několik.

  • Detektor je identifikován hodnotou atributu id (číselník detektorů není součástí této publikace).
  • Samotné měření je identifikováne hodnotou elementu data_id.

Naměřené hodnoty po kategoriích vozidel¶

Jsou poskytovány hodnoty pro:

  • count: počet vozidel
  • occupancy: obsazenost
  • speed: rychlost

Pro každou z těchto typů hodnot je uvedena souhrnná hodnota v elementu all a pak hodnoty po jednotlivých kategoriích vozidel.

Kategorie jsou identifikovány atributem id a jeho význam je popsán ve schématu.

Chybové stavy¶

Pokud je status_generalized roven 1 (viz číselní ve schématu), musí být uvedeny všechny naměřené hodnoty. Jiné hodnoty pak indikují chyby a měřené hodnoty pak chybí.

V elementu error_list jsou uvedeny detekované problémy s poskytováním dat.
