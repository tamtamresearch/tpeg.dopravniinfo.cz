============================
Appendix A: Message examples
============================

restriction.xml - road works, single lane alternate traffic
===========================================================

Publication contains exactly one message about road works with single lane alternate traffic.

.. literalinclude:: ../../samples/restriction.xml
   :language: xml

Root element and publication envelope
-------------------------------------

Root element `d2LogicalModel` declares, that this message is using DATEX II.

Element `exchange` contains provider identifier.

Element `payloadPublication` is an envelope for actual information payload containing general information and one or more elements `situation`, (in our case there is one).

Attribute `xsi:type="SituationPublication"` in element `payloadPublication` determines, that this envelope is of type Situation publication.

Element `feedType` states publication schema by its uri.

Element `publicationTime` states publication time of this publication (file) as a whole.

Element `publicationCreator` identifies creator of the publication.


Description of one specific situation
-------------------------------------

The following element `situation` may be repeated and describes individual separate traffic information. The `situationRecord` element occurs exactly twice in the `situation` element in this case.

The `situationVersionTime` element indicates the time of release of this version of the situation as a whole.

The `headerInformation` element describes how the described information is handled.

The `probabilityOfOccurrence` element with value `riskOf` refers to a planned closure that has not yet occurred.

The `urlLink` element contains a link to a web site where more information about the situation is available.


Event description (what happened)
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

The first element `situationRecord` of type `xsi:type="ConstructionWorks"` contains the description of the cause, i.e. the ongoing construction work `constructionWork`.
The second element `situationRecord` of type `xsi:type="RoadOrCarriagewayOrLaneManagement"` contains a description of the impact, i.e. alternating one-way traffic `singleAlternateLineTraffic`.

Both elements of `situationRecord` are children of the `OperatorActions` class used in this format.

A `complianceOption` with value `mandatory` specifies that this is a binding constraint.

Time validity
^^^^^^^^^^^^^

Estimated time validity is given in a structure `validity` by values of validity interval start `overallStartTime` and validity interval end `overallEndTime`.

Event location
^^^^^^^^^^^^^^

The location of the event described directly in the `groupOfLocations` element by the nested description is of the `xsi:type="Linear"` type, i.e. the linear location type.

In the example, 4 location descriptions are used: 

- tmc location using location tables in the `alertCLinear` element of type `AlertCMethod2Linear`,
- description using stationing in the `linearWithinLinearElement` element,
- description by reference to Global Network section ids in the `globalNetworkLinear` extension, and
- description using the OpenLR routing method in the `openlrExtendedLinear` extension.

All location description methods used express the same location.


Additional content
^^^^^^^^^^^^^^^^^^

The document states time of its first version in `situationRecordCreationTime` and time of current version in `situationRecordVersionTime` for each `situationRecord`.

Element `source` lists the the creator of this information, who is responsible for its accuracy.

The `generalPublicComment` element contains a general description of the event described in the message, usually only in italics. It is a free description of the situation by its creator.
