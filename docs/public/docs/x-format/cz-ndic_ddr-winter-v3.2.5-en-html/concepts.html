Concepts¶

This section highlights main concepts, used by this format. For complete description refer to schemas with definitions in comments.

In this section are provided some snippets of real XML documents. For more examples, see Appendix A: Message examples

XML Document Sections¶

The XML Document is built using following elements and sections:

  • DOC (document root)

  • INF (metadata)

  • MJD (container for messages)

  • MSG (message)

Document root (DOC)¶

The DOC element represents single publication (one instance of information message).

Attribute id provides unique identity to given publication as a whole.

1<?xml version="1.0" encoding="utf-8"?>
2<DOC DataSet="extended" country="CZ" id="{6E6A4889-7F14}" version="3.0">
3  <INF receiver="JB007" sender="JSDI_NDIC" transmission="HTTP">
4    <DAT>
5      <UIRADR structure="4.2" version="522"/>
6    </DAT>
7  </INF>
8  <MJD count="1">

Metadata (INF)¶

INF element provides meta-information about whole publication:

  • reference to GDF 4.0 map data set publisher and version

  • reference to state organization units and areas enumeration UIR-ADR

1  <INF receiver="JB007" sender="JSDI_NDIC" transmission="HTTP">
2    <DAT>
3      <UIRADR structure="4.2" version="522"/>
4    </DAT>
5  </INF>
6  <MJD count="1">

Message container (MJD)¶

Single traffic messages are put into one parent element MJD. It can contain one or more messages.

Traffic Message (MSG)¶

Particular message is represented by a MSG element.

The message itself has following sections:

  • validity MTIME

  • textual description of whole event MTXT

  • weather description MEVT/WCOND

  • road conditions description MEVT/MTNCOND

  • location reference WDEST (where) to enumeration of winter conditions report region

  • reference to a region in Czech Republic MDST

Message identity and version¶

Message itself has unique identifier id and version. Version shall start from 1 and be incremented by each update.

Validity of the message in time is specified by simple time range from start time to end time, but in fact only start time is relevant.

 9    <MSG id="40172-124" planned="false" type="WCOND" version="1">
10      <MTIME format="YYYY-MM-DDThh:mm:ssTZD">
11        <TGEN>2010-02-01T02:58:14+01:00</TGEN>
12        <TSTA>2010-02-01T02:57:00+01:00</TSTA>
13        <TSTO>2010-02-01T14:55:00+01:00</TSTO>
14      </MTIME>

Weather description¶

The MEVT/WCOND element contains set of meteorological parameters (temperature, precipitation, wind, visibility etc.)

17        <WCOND urgency="2">
18          <TEMP from="-8" to="-7" unit="°C"/>
19          <CLD CloudyCode="8" language="CZ">zataženo</CLD>
20          <PREC PrecipitationCode="10" language="CZ">sněhové přeháňky</PREC>
21          <WIND WindCode="2" WindDirectionCode="1" language="CZ">bezvětří</WIND>
22          <VIS VisibilityCode="2" language="CZ">viditelnost bez omezení</VIS>
23          <WTXT language="CZ">Počasí: zataženo, sněhové přeháňky, bezvětří, viditelnost bez omezení</WTXT>
24          <TTXT language="CZ">Teplota od -8 do -7°C</TTXT>
25        </WCOND>

Location description¶

DDR (Winter) is using references to areas, from which are information provided.

There are following location referencing methods used:

  • Reference to enumeration of report regions (see Enumeration of report regions)

  • Coordinate of centre of report region

  • Reference to region in Czech Republic using enumeration UIR-ADR.

39      <WDEST NewsRegionCode="124" NewsRegionName="okres Havlíčkův Brod" coordsystem="S-JTSK">
40        <COORD x="-678806" y="-1095819"/>
41      </WDEST>
42      <MLOC/>
43      <MDST>
44        <DEST CountryName="Česká republika" RegionCode="108" RegionName="Vysočina kraj" TownShip=" Havlíčkův Brod " TownShipCode="3601"/>
45      </MDST>
