Annex A: Examples of messages¶

samples/fcd-compatibility-mode.xml - FCD data on 1 segment¶

The sample contains FCD information for one physical location (route with identifier 6D476A44-AA44-4947-AA4B-7C4DD79BB67E).

The routes are defined in another publication of type PredefinedLocationsPublication, which is not described in detail here.

<?xml version="1.0" encoding="utf-8"?>
<d2LogicalModel modelBaseVersion="2" xmlns="http://datex2.eu/schema/2/2_0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://datex2.eu/schema/2/2_0 ../schema/schema.xsd">
  <exchange>
    <supplierIdentification>
      <country>cz</country>
      <nationalIdentifier>ŘSD</nationalIdentifier>
    </supplierIdentification>
  </exchange>
  <payloadPublication lang="cs" xsi:type="ElaboratedDataPublication">
    <feedType>x-format:cz-ndic_d2-fcd-v1.0</feedType>
    <publicationTime>2017-11-18T00:00:06+01:00</publicationTime>
    <publicationCreator>
      <country>cz</country>
      <nationalIdentifier>ŘSD</nationalIdentifier>
    </publicationCreator>
    <periodDefault>300</periodDefault>
    <timeDefault>2017-11-17T23:59:35+01:00</timeDefault>
    <headerInformation>
      <confidentiality>noRestriction</confidentiality>
      <informationStatus>real</informationStatus>
    </headerInformation>
    <!-- information from one segment -->
    <!-- traffic status element -->
    <elaboratedData>
      <source>
        <sourceIdentification>ndic-secid-1-01</sourceIdentification>
      </source>
      <basicData xsi:type="TrafficStatus">
        <measurementOrCalculationTime>2017-11-17T23:57:35+01:00</measurementOrCalculationTime>
        <pertinentLocation xsi:type="LocationByReference">
          <predefinedLocationReference id="6D476A44-AA44-4947-AA4B-7C4DD79BB67E" targetClass="PredefinedLocation" version="0"/>
        </pertinentLocation>
        <!-- traffic status information -->
        <trafficStatus numberOfInputValuesUsed="9" supplierCalculatedDataQuality="1">
          <trafficStatusValue>congested</trafficStatusValue>
        </trafficStatus>
        <!-- extension: traffic level and queue information -->
        <trafficStatusExtension>
          <ndicFcdExtension>
             <trafficLevel>
              <trafficLevelValue>level4</trafficLevelValue>
            </trafficLevel>
            <queueInformation>
              <queue>
                <queueLength>148</queueLength>
                <!-- start of the queue (tail) at the 0.4 of the length of the segment -->
                <fromPoint>0.4</fromPoint>
                <!-- end of the queue (head) at the 0.9 of the length of the segment -->
                <toPoint>0.9</toPoint>
              </queue>
            </queueInformation>
          </ndicFcdExtension>
        </trafficStatusExtension>
      </basicData>
    </elaboratedData>
    <!-- speed information element -->
    <elaboratedData>
      <source>
        <sourceIdentification>ndic-secid-1-01</sourceIdentification>
      </source>
      <basicData xsi:type="TrafficSpeed">
        <measurementOrCalculationTime>2017-11-17T23:57:35+01:00</measurementOrCalculationTime>
        <pertinentLocation xsi:type="LocationByReference">
          <predefinedLocationReference id="6D476A44-AA44-4947-AA4B-7C4DD79BB67E" targetClass="PredefinedLocation" version="0"/>
        </pertinentLocation>
        <!-- average vehicle speed information -->
        <averageVehicleSpeed>
          <speed>38</speed>
        </averageVehicleSpeed>
      </basicData>
    </elaboratedData>
    <!-- travel time element -->
    <elaboratedData>
      <source>
        <sourceIdentification>ndic-secid-1-01</sourceIdentification>
      </source>
      <basicData xsi:type="TravelTimeData">
        <measurementOrCalculationTime>2017-11-17T23:57:35+01:00</measurementOrCalculationTime>
        <pertinentLocation xsi:type="LocationByReference">
          <predefinedLocationReference id="6D476A44-AA44-4947-AA4B-7C4DD79BB67E" targetClass="PredefinedLocation" version="0"/>
        </pertinentLocation>
        <!-- travel time information -->
        <vehicleType>anyVehicle</vehicleType>
        <travelTime numberOfInputValuesUsed="9" supplierCalculatedDataQuality="1">
          <duration>549</duration>
        </travelTime>
        <!-- historical data from fcd vehicles database -->
        <freeFlowTravelTime numberOfInputValuesUsed="9999" supplierCalculatedDataQuality="1">
          <duration>259</duration>
        </freeFlowTravelTime>
        <!-- free flow vehicle speed information -->
        <freeFlowSpeed supplierCalculatedDataQuality="0.8">
          <speed>50</speed>
        </freeFlowSpeed>
        <!-- extension: quality parameters and number of vehicles on the segment -->
        <travelTimeDataExtension>
            <qualityParameters>
              <supplierCalculatedDataQuality>1</supplierCalculatedDataQuality>
              <numberOfFcdVehicles>
                <vehicleGroup>
                  <count>9</count>
                  <vehicleType>anyVehicle</vehicleType>
                </vehicleGroup>
              </numberOfFcdVehicles>
            </qualityParameters>
         </travelTimeDataExtension>
      </basicData>
    </elaboratedData>
    <!-- information from another segment follows -->
  </payloadPublication>
</d2LogicalModel>

Root element and the publication envelope¶

The root element d2LogicalModel declares that this is a message using DATEX II.

The exchange element contains the provider identifier.

The payloadPublication element is an envelope for the information itself (payload), containing primarily elaboratedData elements, of which there may be many (seven in this example).

The xsi:type=”ElaboratedDataPublication” attribute in the payloadPublication element specifies that this envelope is of type ElaboratedData.

The feedType element specifies the type of publication, by the uri of the schema.

The publicationTime element specifies the time of the publication.

The publicationCreator element identifies the creator of the publication.

The timeDefault element indicates the time of validity of the calculated traffic statuses, this is the time of calculation of all FCD values given in the report.

General characteristics of the elaboratedData elements¶

The following three elaboratedData elements list the individual characteristics of the FCD measurements. However, a few characteristics are similar/same for all elaboratedData elements, they are listed below.

Location reference¶

The id=”6D476A44-AA44-4947-AA4B-7C4DD79BB67E” attribute in the predefinedLocationReference elements of each elaboratedData record refers to a predefined segment (location) with a given identifier, and the version=”0” attribute specifies which version of the location reference it is.

Temporal validity¶

The start of validity of the given status values is specified together for all records in the timeDefault element (2017-11-17T23:59:35+01:00).

The last time of occurrence of vehicles, data sources, (2017-11-17T23:57:35+01:00) on the measured segment is given in the measurementOrCalculationTime elements. This value may be different for each segment.

Measurement quality¶

The measurement given in the qualityParameterssupplierCalculatedDataQuality elements and the optional supplierCalculatedDataQuality attributes of the calculated values are set to 1. It means “reliable”. This value should be the same for all actual measurement values on the segment.

Number of vehicles on the segment¶

The element numberOfFcdVehicles and optional attributes numberOfInputValuesUsed are set to 9. It means there were 9 FCD vehicle probes on the given segment (route) at the time of measurement (300 s). This value should be the same for all the actual measurement values on a given segment.

Source identification¶

The optional sourceIdentification element in elaboratedData elements may provide an alternative route identification. The value is used to facilitate identification in addition to the location identifier.

Description of traffic levels and queues¶

The first element elaboratedData describes the calculated traffic status and its statistical data.

The nested element basicData indicates by its attribute xsi:type=”TrafficStatus” that the values shall be interpreted as “traffic density”.

The trafficStatus element indicates the level of traffic on a four-level scale, as “congested”.

The following portions of the elaboratedData element are Level B extensions of the DATEX II model.

Traffic levels¶

The trafficLevel element lists the level of traffic in the level sub-element by enumeration type, in this case with a value of level4, i.e. level 4 (equivalent of congested).

This value also indicates the presence of a traffic jam.

Queue length¶

The queue information is encapsulated in the queueInformation element. Individual queues are listed in the queue element. In the queueLength sub-element the actual value of the length of one queue on a given route (place) at a given time = 148 m.

The sub-elements fromPoint and toPoint give the start (0.4 of the segment length) and end (0.9) of the segment length) of the queue in relative coordinates.

Description of the measured speed¶

The second element elaboratedData describes the computed traffic flow speed with its statistical data.

The nested element basicData indicates by its attribute xsi:type=”TrafficSpeed” that the values shall be interpreted as “traffic speed”.

Traffic speed¶

The averageVehicleSpeed element gives the actual average traffic speed of 38 km/h.

Description of travel time and speed¶

The third element elaboratedData describes the calculated actual and historical travel times and speed of the continuous traffic flow with their statistical data.

The nested element basicData indicates by its attribute xsi:type=”TravelTimeData” that the values shall be interpreted as “travel times”.

The vehicleType element indicates that the following values apply to all vehicles anyVehicle.

Travel time¶

The travelTime element gives the average actual travel time of 549 s.

The freeFlowTravelTime element gives the average travel time in continuous operation as 259 s.

The normallyExpectedTravelTime element gives the average travel time at a given location at a given time as 348 s, so this is an abnormal condition.

Speed¶

The freeFlowSpeed element gives the average speed in in freely flowing traffic at 50 km/h.

samples/fcd-compatibility-mode-no-data.xml - no FCD data on 1 segment¶

The sample contains FCD information for the same location as the previous sample. The difference between the samples is the unavailability of data to calculate status information.

  <payloadPublication lang="cs" xsi:type="ElaboratedDataPublication">

    <publicationTime>2017-11-18T00:00:06+01:00</publicationTime>
    <publicationCreator>
      <country>cz</country>
      <nationalIdentifier>ŘSD</nationalIdentifier>
    </publicationCreator>
    <periodDefault>300</periodDefault>

In the sections below, we only list the differences from the previous sample.

General characteristics of elaboratedData elements¶

The following three elaboratedData elements list the individual characteristics of the FCD measurements. However, a few characteristics are similar/same for all elaboratedData elements, they are listed below.

Temporal validity¶

The start of validity of the given status values is specified together for all records in the timeDefault element is 2017-11-17T23:59:35+01:00.

The last time of occurrence of vehicles, data sources, on the measured segment listed in the measurementOrCalculationTime element is 2017-11-17T22:57:35+01:00, about an hour ago.

Measurement quality¶

Value in qualityParameterssupplierCalculatedDataQuality and optional supplierCalculatedDataQuality is 0. It means “not-reliable”.

Number of vehicles on the segment¶

Value in numberOfFcdVehicles and optional attribute numberOfInputValuesUsed is 0. It means there were NO FCD vehicle probes on the given segment at the time of measurement (300 s).

Description of traffic levels and queues¶

The first element elaboratedData describes the calculated traffic status, traffic level and queues.

The related element elaboratedData is not provided.

Description of the measured speed¶

The second element elaboratedData describes the calculated traffic flow speed.

The related element elaboratedData is not provided.

Description of travel time and speed¶

The third element elaboratedData describes the calculated actual and historical travel times and speed of the continuous traffic flow with their statistical data.

Travel time¶

Only the freeFlowTravelTime element is given, indicating an average travel time in a free flow traffic conditions of 259 s.

Speed¶

Only the freeFlowSpeed element is given, giving an average speed in a free flow traffic conditions at 50 km/h.
