Příloha B: Reprezentativní XML Schéma RELAX NG¶

K popisu struktury XML Je použito RELAX NG schéma v kompaktním formátu. Doporučujeme zkusit dané schéma přečíst, protože se nám tato forma jeví jako nejkompaktnější a nejčitelnější z dostupných typů schémat.

Pokud však trváte na použití W3C XML Schema 1.0, přikládáme i tento typ schématu, které je vytvořeno konverzí z RELAX NG.

RELAX NG Compact Syntax Schema (RNC)¶

RNC schéma popisuje očekávané struktury XML dokumentu.

Kořenový element je uveden pomocí start =.

Všechny další <SomeName> = elementy definují tzv. vzory.

V našem schématu jsou následující sekce:

  • prolog (deklarace jmenného prostoru)

  • vložení jiných schéma (není využito v tomto případě)

  • definice kořenového elementu (start =)

  • vzory pro elementy

  • vzory pro atributy

  • vzory pro datové typy

Komentáře ## poskytují více detailů k významu jednotlivých částí.

WinterConditions.rnc - kořen schématu¶

default namespace = ""

start =

## Zapouzdřuje celý dokument pro výměnu informací
element DOC {

    ## Verze XML definice
    attribute version { type-formatVersion },

    ## Jednoznačný identifikátor dokumentu
    ## GUID může mít různý tvar (6-4, 6-4-4-4-6, 8-4-4-4-12, číslo, ...), vždy je
    ## však unikátní.
    attribute id { text },

    ## Kód země, ve které se události obsažené ve zprávě nachází. Pozn.:
    ## V jedné zprávě se nesmí nacházet událostí z více státu.
    attribute country { type-country },

    ## Použitá datová sada. Datová sada seskupuje údaje související s informací.
    ##
    ## Varianty "basic" a "custom" vyřazeny.
    attribute DataSet { "extended" },

    ## Informace o subjektech, které si vyměnují XML dokument
    element INF {

        ## Zkratka systému, ve kterém byla dopravní informace pořízena
        ##
        ## Například: "JSDI_NDIC"
        attribute sender { text },

        ## Zkratka systému/subjektu, pro který je zpráva určena napr. HZS
        attribute receiver { text },

        ## Typ přenosu použitý k předání dokumentu
        attribute transmission { "HTTP" | "SMTP" | "FTP" },

        ## Informace o verzích jednotlivých datových sad
        element DAT {
        
            ## Obsahuje informace o lokační tabulce
            # TODO: Není obsažen v dokumentaci
            # TODO: Některé vzorky jej neobsahují
            element LOCT {
            
                ## Verze lokační tabulky
                attribute version { type-AlertCLocationTableVersion },
                
                ## Číslo lokační tabulky
                attribute number { xsd:positiveInteger },
                
                ## Číslo země podle číselníku Alert-C
                attribute country { xsd:positiveInteger }
            }?,
            
            ## Informace o datové sadě sítě komunikací, na kterou jsou lokalizovány dopravní události
            ##
            ## Povinný v případě, že zpráva obsahuje lokalizaci na síť komunikací
            element SNET {
                 
                ## Informace o typu použité sítě komunikací
                ## "SN" = streetnet "GN" = GlobalNetwork
                attribute type { type-snet-type },

                ## Informace o verzi sítě
                attribute version { type-networkVersion },
                
                ## Země, pro kterou je vytvořena síť komunikací 
                attribute country { "CZ" }
            }?,
            
            ## Informace o datové sadě UIRADR
            # Dokumentace hovoří o elementu UIADR (chyba z přehlédnutí)
            # Dokumentace dovoluje uvézt pouze v případě, že zpráva obsahuje české události s vyplněnou informací o
            # místu události v souladu s republikovým číselníkem UIRADR.
            element UIRADR {
            
                ## Informace o verzi struktury
                attribute structure { text },
                
                ## Informace o verzi dat
                attribute version { type-nonZeroDecimal },
                
                ## Datum vydání verze dat
                attribute date { xsd:date }?
            }
        }
    },
    
    ## Zapouzdřuje veškeré informace o dopravních informacích
    element MJD {
    
        ## Počet dopravních informací, které jsou zapouzdřeny v MJD
        attribute count { xsd:positiveInteger },
        
        ## Představuje jednu zprávu v dokumentu
        element-MSG*
    }
}

element-MSG =

## Představuje jednu zprávu v dokumentu
element MSG {

    ## Jednoznačný identifikátor zprávy
    ##
    ## Formát jednotlivých ID se liší podle poskytovatele události.
    attribute id { text },
    
    ## Číslo verze zprávy
    ##
    ## Nová = 1, aktualizace, vždy +1
    # TODO: Existují zprávy s version = "0"
    attribute version { xsd:nonNegativeInteger },
    
    ## Typ poskytované informace
    attribute type { "WCOND" },
    
    ## Typ poskytované informace z hlediska jejího charakteru
    ##
    ## Rozlišujeme událost na:
    ## plánovanou z hlediska úředního povolení / skutečný stav
    attribute planned { xsd:boolean },

    ## Časové informace o zprávě
    element MTIME {
    
        ## Popisuje (konstantní) formát datetime.
        attribute format { "YYYY-MM-DDThh:mm:ssTZD" },

        ## Čas kdy byla zpráva vygenerována
        element TGEN { xsd:dateTime },
        
        ## Čas od kdy zpráva platí nebo bude platná
        element TSTA { xsd:dateTime },

        ## Čas do kdy bude zpráva platná
        element TSTO { xsd:dateTime }
    },
    
    ## Kompletní textový popis události vyjadřující KDE, CO, KDY, případně JAK dlouho
    element MTXT {
    
        ## Jazyk, ve kterém je zpráva napsaná 
        attribute language { type-language-cz },
        
        ## Popis
        text
    },
    
    ## Informace o události 
    element-MEVT,

    ## Informace o místu a názvu zpravodajské oblasti 
    element-WDEST,
    
    ## Informace o místu události
    (element-MLOC | element-MLOC-empty)?,
    
    ## Informace o místě události v souladu s republikovým číselníkem UIRADR
    # Dokumentace povoluje vynechat v případě, že zpráva neobsahuje české události s vyplněnou informací o
    # místu události v souladu s republikovým číselníkem UIRADR
    element-MDST?
}
