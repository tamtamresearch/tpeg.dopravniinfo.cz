Appendix A: Message examples¶

meteo-data.xml - weather data measured at one equipment¶

Publication contains one record for one equipment capable to measure several weather related characteristics.

Equipment details (capabilities and location) are defined in different publication of type MeasurementSiteTablePublication, which is not described here in detail.

The equipment, is identified by a measurement site table reference id=”CZ_DRU_MST_WEA” (with version), and by measurement site reference id=”12”, (with version).

<?xml version="1.0" encoding="utf-8"?>
<d2LogicalModel modelBaseVersion="2" xmlns="http://datex2.eu/schema/2/2_0" xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <exchange>
    <supplierIdentification>
      <country>cz</country>
      <nationalIdentifier>DRU</nationalIdentifier>
    </supplierIdentification>
  </exchange>
  <payloadPublication lang="cs" xsi:type="MeasuredDataPublication">
    <feedType>x-format:cz-ndic_d2-weather-data-v1.0</feedType>
    <publicationTime>2018-03-21T15:59:20+02:00</publicationTime>
    <publicationCreator>
      <country>cz</country>
      <nationalIdentifier>DRU</nationalIdentifier>
    </publicationCreator>
    <measurementSiteTableReference id="CZ_DRU_MST_WEA" targetClass="MeasurementSiteTable" version="1"/>
    <headerInformation>
      <confidentiality>noRestriction</confidentiality>
      <informationStatus>real</informationStatus>
    </headerInformation>
    <siteMeasurements>
      <measurementSiteReference id="12" targetClass="MeasurementSiteRecord" version="1"/>
      <measurementTimeDefault>2018-03-21T15:59:20+02:00</measurementTimeDefault>
      <!--precip_type + precipitation -->
      <measuredValue index="1">
        <measuredValue>
          <basicData xsi:type="PrecipitationInformation">
            <precipitationDetail>
              <precipitationType>rain</precipitationType>
              <precipitationIntensity>
                <millimetresPerHourIntensity>10</millimetresPerHourIntensity>
              </precipitationIntensity>
            </precipitationDetail>
          </basicData>
        </measuredValue>
      </measuredValue>
      <!--road_status -->
      <measuredValue index="2">
        <measuredValue>
          <basicData xsi:type="RoadSurfaceConditionInformation">
            <weatherRelatedRoadConditionType>surfaceWater</weatherRelatedRoadConditionType>
            <roadSurfaceConditionMeasurements>
              <waterFilmThickness>
                <floatingPointMetreDistance>0.01</floatingPointMetreDistance>
              </waterFilmThickness>
            </roadSurfaceConditionMeasurements>
          </basicData>
        </measuredValue>
      </measuredValue>
      <!--road_temp 1 + freez_temp-->
      <measuredValue index="3">
        <measuredValue>
          <basicData xsi:type="RoadSurfaceConditionInformation">
            <roadSurfaceConditionMeasurements>
              <roadSurfaceTemperature>
                <temperature>8.7</temperature>
              </roadSurfaceTemperature>
              <protectionTemperature>
                <temperature>0</temperature>
              </protectionTemperature>
            </roadSurfaceConditionMeasurements>
          </basicData>
        </measuredValue>
      </measuredValue>
      <!--road_temp 2 + freez_temp-->
      <measuredValue index="4">
        <measuredValue>
          <measurementEquipmentFault>
            <faultLastUpdateTime>2018-03-21T15:59:20+02:00</faultLastUpdateTime>
            <measurementEquipmentFault>noDataValuesAvailable</measurementEquipmentFault>
          </measurementEquipmentFault>
        </measuredValue>
      </measuredValue>
      <!--air_temp + dewpoint -->
      <measuredValue index="6">
        <measuredValue>
          <basicData xsi:type="TemperatureInformation">
            <temperature>
            <airTemperature>
              <temperature>8.1</temperature>
            </airTemperature>
            <dewPointTemperature>
              <temperature>0.1</temperature>
            </dewPointTemperature>
            </temperature>
          </basicData>
        </measuredValue>
      </measuredValue>
      <!--humidity -->
      <measuredValue index="7">
        <measuredValue>
          <basicData xsi:type="HumidityInformation">
            <humidity>
              <relativeHumidity>
                <percentage>57</percentage>
              </relativeHumidity>
            </humidity>
          </basicData>
        </measuredValue>
      </measuredValue>
      <!--wind_dir +  wind_sp-->
      <measuredValue index="8">
        <measuredValue>
          <basicData xsi:type="WindInformation">
            <wind>

              <windDirectionBearing>

              </windDirectionBearing>
            </wind>

Root element and publication envelope¶

Root element d2LogicalModel declares, that this message is using DATEX II.

Element exchange contains provider identifier.

Element payloadPublication is an envelope for actual information payload containing general information, measurement site table reference measurementSiteTableReference and one element siteMeasurements.

Attribute xsi:type=”MeasuredDataPublication” in element payloadPublication determines, that this envelope is of type Measured data publication.

Element feedType states publication schema by its uri.

Element publicationTime gives publication time for the whole publication which might be the time, for which were all traffic status values calculated (start of validity).

Element publicationCreator identifies creator of the publication.

Description of weather data at one place¶

The element siteMeasurements contains one or more measurements referenced to a particular road equipment measurementSiteReference. The measurementSiteReference is identified by id and version.

measurementTimeDefault element gives time of validity for the set of measurement within one measurement site. It is also the time, when measured weather data were created (i.e. the end of the measurement period).

Following element measuredValue can be repeated and describes a measured weather data for a particular measurement site.

The equipment (weather station) has 9 capabilities defined (defined in MeasurementSiteTablePublication). They are:

  • precipitation,

  • road surface conditions (3x),

  • temperature,

  • humidity,

  • visibility and

  • wind.

The road surface conditions is there 3 times (indexes 2-4). First one defines overall characteristics and is used to convey road status. Remaining two are road lane related measurements (one for each lane) the latter form the two is in faulty state measurementEquipmentFault.

Nested element basicData contains the actual measured value of the given type.

Time validity¶

Start of validity for measured data is stated by mandatory element measurementTimeDefault.
