Root element and publication envelope¶

Root element d2LogicalModel declares, that this message is using DATEX II.

Element exchange contains provider identifier.

Element payloadPublication is an envelope for actual information payload containing general information and one or more elements predefinedLocationContainer.

Attribute xsi:type=”PredefinedLocationsPublication” in element payloadPublication determines, that this envelope is of type predefined location publication.

Element feedType states publication schema by its uri.

Element publicationTime states publication time of this publication as a whole.

Element publicationCreator identifies creator of the publication.

Description of predefined locations¶

The predefined location is described in the predefinedLocationContainer element by a directly nested description.

The example publication describes 3 predefined locations with the identifiers id=”TS25760T25761”, id=”TS25761T25760” and id=”TS27072T27073”. All locations contain 3 methods describing the same slice of reality. These are sections of the FCD data location network.

Position description methods¶

The predefinedLocationName element contains the name of the location, in this case the same as the identifier.

The location element then contains the location description itself. The xsi:type=”Linear” attribute indicates that this is a linear type of location.

In the example, there are 3 location descriptions used for all predefinedLocationContainer:

  • tmc location using location tables in the alertCLinear element of type AlertCMethod2Linear,

  • description using references to Global Network section ids in the globalNetworkLinear extension, and

  • description using the OpenLR routing method in the openlrExtendedLinear extension.
