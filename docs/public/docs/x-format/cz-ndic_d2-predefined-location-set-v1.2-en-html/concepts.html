Concepts¶

Described format allows publishing of predefined locations used for referencing of status information (travel times, level of service, tunnel status, parking occupancy, weather station reports etc.).

Publication type PredefinedLocationPublication¶

Format is based on DATEX II publication of type PredefinedLocationPublication [16157-3]. The profile uses most of data structures, provided by this type of publication and extends it by proprietary location referencing.

Predefined location description¶

Each predefinedLocationContainer of type xsi:type=”PredefinedLocation” contains a description of the location of one real world location. There are as many predefinedLocationContainer elements in the publication as there are predefined locations. The uniqueness of the predefined locations in this publication is guaranteed by the unique identifier and version.

20    <predefinedLocationContainer id="96680652-f696-4b53-b97e-88d16f1943ce" version="1" xsi:type="PredefinedLocation">

In a publication, it is possible to use predefinedLocationContainer of type xsi:type=”PredefinedNonOrderedLocationGroup” to describe multiple non-continuos locations together, each separate location is then contained in nested predefinedLocation elements.

20    <predefinedLocationContainer xsi:type="PredefinedNonOrderedLocationGroup" id="73D74C70-BFA8-408F-86CD-49B3689D9F5D" version="1">
21      <predefinedLocation id="dde51831-2f37-4f07-8429-286a9d08106c" version="1">

The location can be described by a point or a short line segment on the road network. Locations of type area are not used.

The position description methods are further described in the chapter “Location description”.

Time validity¶

The publication start of validity is defined by publicationTime for all predefined locations in the publication. No other time record is contained in the publication.

Note

In case of new publication due to changes to predefined location set, old locations remains the same, new locations get new id and updated locations got its version increased.

10    <feedType>x-format:cz-ndic_d2-predefined-location-set-v1.2</feedType>
11    <publicationTime>2023-08-15T10:09:22+02:00</publicationTime>

Location referencing¶

All locations in this publication are predefined and each is contained within element PrefefinedLocationContainer globally identified by its identifier and version (these are used to reference the location from other publications). Element location (descendant of NetworkLocation) with type Point or Linear contains description of one physical location in one or more location referencing methods.

Three location description methods for a linear location and four location description methods for point location can be used to describe the location. The method use depends on method availability at the location. At least one location description shall be used.

Location description by means of ALERT-C¶

The method describes the position using ALERT-C codes, class alertCLinear of type AlertCMethod2Linear is used. The use of this method depends on the presence of an ALERT-C location at a given location.

The alertCLinear method uses a primary location (alertCMethod2PrimaryPointLocation element), a secondary location (alertCMethod2SecondaryPointLocation element) to describe the event. Of course, the method also contains elements allowing to identify a specific location table.

The primary location is, according to the ALERT-C principle, the place where the accident occurred or where the driver leaves the problem area. The secondary location is the end of the convoy or where the driver first enters the problem area.

In order to find out the direction of the situation, the alertCDirectionCoded element is used, which takes the value positive if the direction of the traffic flow agrees with the coding of the given route.

Important

WARNING, the meaning of this attribute is therefore exactly the opposite of the direction attribute in RDS-TMC, which is used to find the secondary location.

The alertCLinear element may occur multiple times in the situation description for long events that must be described by more than one ALERT-C path. This is because ALERT-C only allows locations to be chained within a single path.

Note

The alertCLinear method is also used to describe a point location (even the inside of an intersection or tunnel, i.e. a short segment). When primary and secondary locations are the same. The direction can be read from the alertCDirectionCoded element).

111      <location xsi:type="Linear">
112        <alertCLinear xsi:type="AlertCMethod2Linear">
113          <alertCLocationCountryCode>CZ</alertCLocationCountryCode>
114          <alertCLocationTableNumber>25</alertCLocationTableNumber>
115          <alertCLocationTableVersion>9.0</alertCLocationTableVersion>
116          <alertCDirection>
117            <alertCDirectionCoded>negative</alertCDirectionCoded>
118          </alertCDirection>
119          <alertCMethod2PrimaryPointLocation>
120            <alertCLocation>
121              <specificLocation>8973</specificLocation>

Location description using mileage¶

The method uses linearWithinLinearElement for the description of the position using mileage. The roadNumber element describes the number of the affected road, the directionRelativeOnLinearSection element describes the direction of the affected road, and the fromPoint and toPoint elements describe the distance of the start and end of the event from the start of the road.

140      <location xsi:type="Linear">
141        <linearWithinLinearElement>
142          <directionRelativeOnLinearSection>aligned</directionRelativeOnLinearSection>
143          <linearElement>
144            <roadNumber>10</roadNumber>
145          </linearElement>
146          <fromPoint xsi:type="DistanceFromLinearElementStart">
147            <distanceAlong>106797</distanceAlong>
148          </fromPoint>
149          <toPoint xsi:type="DistanceFromLinearElementStart">
150            <distanceAlong>107507</distanceAlong>
151          </toPoint>
152        </linearWithinLinearElement>
153      </location>

The method for description of a point event PointAlongLinearElement differs by simply describing one distance from the beginning of the identified road instead of two.

163      <location xsi:type="Point">
164        <pointAlongLinearElement>
165          <directionRelativeAtPoint>aligned</directionRelativeAtPoint>
166          <linearElement>
167            <roadNumber>D1</roadNumber>
168          </linearElement>
169          <distanceAlongLinearElement xsi:type="DistanceFromLinearElementStart">
170            <distanceAlong>4500</distanceAlong>
171          </distanceAlongLinearElement>
172        </pointAlongLinearElement>
173      </location>

Location description using the Global Network extension¶

Native method of position input in NDIC using Extension of DATEX II model of level B.

The method describes, in the globalNetworkLinear element, the location using the segments linearWithinLinearGNElement of the Global Network (GN) road network, identified by the networkVersion version. Each segment contains a sequence, an identifier referring to the GN, and a relative truncation (for the first and last segment). Furthermore, the position is bounded by the elements startPoint and endPoint in coordinates (here S-JTSK).

Note

The globalNetworkLinear method is also used to describe a point location (even the inside of an intersection or tunnel, i.e. a short segment). The startPoint and endPoint positions contain the same coordinate.

Location description using OpenLR extension¶

The OpenLR method in the openlrExtendedLinear element, uses a routing algorithm to describe the location and thus allows on-the-fly localization.

The method uses the reference points openlrLocationReferencePoint and openlrLastLocationReferencePoint containing coordinates, road direction, and a road type. These reference points are used as the start and end to find the road which is further refined using offsets to the actual extent of the event. In case 2 points are not enough the method allows to use intermediate points.

Note

To describe a point location (even the inside of an intersection or tunnel, i.e. a short segment) the openlrExtendedLinear method is also used, when offsets refer to a segment with zero length.

200      <location xsi:type="Linear">
201        <linearExtension>
202          <openlrExtendedLinear>
203            <firstDirection>
204              <openlrLocationReferencePoint>
205                <openlrCoordinate>
206                  <latitude>50.765282088427412</latitude>
207                  <longitude>15.337521544012871</longitude>
208                </openlrCoordinate>
209                <openlrLineAttributes>

213                </openlrLineAttributes>

Location description using Coordinates¶

The method uses geographic coordinates to describe a situation location in element pointByCoordinates using WGS-84 coordinate system.

183      <location xsi:type="Point">
184        <pointByCoordinates>
185          <pointCoordinates>
186            <latitude>50.00998</latitude>
187            <longitude>14.54135</longitude>
188          </pointCoordinates>
189        </pointByCoordinates>
190      </location>
