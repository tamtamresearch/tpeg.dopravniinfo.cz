========
Concepts
========

Described format allows publishing of predefined equipment data used to reference status oriented events (detector data, weather data etc.).

Publication type MeasurementSiteTablePublication
================================================

Format is based on DATEX II publication of type `MeasurementSiteTablePublication`
[16157-5]_. The profile uses most of data structures, provided by this type of publication and extends it by proprietary location referencing.

Predefined equipment description (measurementSiteTable and measurementSiteRecord)
=================================================================================

`MeasurementSiteTablePublication` requires description by means of class `measurementSiteTable`, which aggregates instances of class `measurementSiteRecord`. `measurementSiteTable` also name of the respective table.

Element `measurementSiteTable` describes a particular type of sites (usually), more particular types of sites or more infrastructure operators means more `measurementSiteTable` elements. Since the equipment described may change over time, the element is versioned and has unique identifier. Each `measurementSiteTable` element contains one or more `measurementSiteRecord` elements.

Textual description of each `measurementSiteRecord` could be located within a  `measurementSiteName` element.

Each `measurementSiteRecord` element is describing different measuring equipment with different measuring capabilities `measurementSpecificCharacteristics` i.e. measuring of traffic flow, speed, density, etc. The capabilities of `measurementSiteRecord` element might change in time, so the element is versioned and has an identifier unique within the scope of one measrement table.

The capabilities `measurementSpecificCharacteristics` of the measuring equipment in particular includes:

- humidity,
- individual vehicle measurements (headway, speed, etc.),
- pollution,
- precipitation,
- pressure,
- radiation,
- road surface conditions,
- temperature,
- traffic concentration,
- traffic flow,
- traffic headway,
- traffic speed,
- traffic status,
- travel time,
- visibility and
- wind.

Some `measurementSpecificCharacteristics`, where it matters can be further specified by positional arguments and vehicle specification (i.e. traffic flow of heavy vehicles, road temperature in second lane). Each capability is indexed (uniquely within one equipment). The index is used for capability referencing from the dynamic publication.

Equipment location description (point or linear segment)
========================================================

Each equipment location is specified within the element `measurementSiteLocation`.

*Location* is a point or linear segment on road network. Location type area is not used. The locations may be described by means of several methods, locations are described in line (not referenced to another publication)

   :language: xml
   :linenos:
   :lineno-match:

   :end-before: <!-- location description end -->

The position description methods are further described in the chapter "Location description".

=============
Time validity
=============

The publication start of validity is defined by `publicationTime` for all predefined equipment in the publication. No other time record is contained in the publication. In case of new publication due to changes to predefined equipment set, existing equipment remains the same, new equipment get new id and updated equipment got its version increased.

====================
Location referencing
====================

All  equipment predefined in this publication has its location set up within element `measurementSiteLocation` with type `Point` or `Linear` that contains description of the equipment physical location in one or more location referencing methods.

.. literalinclude:: ../../samples/measurement_site_table.xml
   :language: xml
   :linenos:
   :lineno-match:
   :start-after: <!-- location description start -->
   :end-before: <!-- location description end -->

Three location description methods for a linear location and four location description methods for point location can be used to describe the location. The method use depends on method availability at the location. At least one location description should be used.
